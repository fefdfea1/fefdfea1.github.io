Next Js 쇼핑몰 제작 프로젝트 레이아웃 구성

posted by fefdfea,
June 21, 2024

레이아웃 제작

CSS 사용중 문제

css를 사용하고 있던중 간과하고 있던 사실이 하나 있었는데 React가 아니라 Next js를 사용하고 있다는 사실을 잊어버리고 있었다. 현재 필자는 css-in-js 방식으로 css 스타일을 주려고 하였지만 emotion은 런타임 환경에서 동작하여 스타일을 주는 방식이기 때문에 서버사이드 방식을 채택하는 Next JS에서 사용에 한계가 있다. 사용이 불가능 한 것은 아니지만 쌩 HTML이 보이고 그다음 스타일이 동적으로 입혀지기 때문에 굉장히 이상하게 보인다. 이 문제에 대해서는 다음에 다루도록 하겠다. 우선 필자는 panda css를 사용하여 작업한다는 것을 알아두자

판다 css

ai 사용을 안하는이유

이번 포스팅에는 본격적으로 레이아웃을 제작할 예정인데 원래 사용하려한 생성형 ai의 디자인을 사용하기가 어려울 것 같아 핀터레스트의 디자인 하나를 가져와 진행할 예정이다.

위의 이미지는 생성형 ai가준 디자인인데 ( 왼쪽은 uizard 오른쪽은 Framer ) 왼쪽은 쇼핑몰 디자인이 요즘 같은 느낌의 디자인이 아니라 사용하고 싶지가 않았으며 오른쪽은 쇼핑몰 조차도 아니다. 그래서 생성형 ai는 사용하지 않을 예정이다

디자인 찾기

핀터레스트의 쇼핑몰 디자인을 찾던중 아래와 같은 디자인을 찾았는데 간단한 쇼핑몰 프로젝트에 사용하기 좋게 심플하여 이걸 사용하고자 한다.

본격적인 레이아웃

우선 구성은 Header, Main , Footer 이렇게 3가지로 나누어 진행을 할 예정이고 이번 포스팅에서는 너무 길어질 수도 있으므로 Header 레이아웃 구성까지만 하고자 한다. 각 영역들은 아래의 이미지와 같이 나누었다.

Header

우선 레이아웃을 제작하기 위한 컴포넌트로 제작할 부분을 따로 제작해보도록 하자 Header를 보면 Navigation 부분이 상당히 반복되는 것을 볼 수 있다. 그러니 저 부분을 컴포넌트로 분리하고 사용해보자

Nav 컴포넌트 제작

컴포넌트 제작을 위해 다음과 같이 component 폴더를 하나 제작하도록 하겠다.

필자는 다음과 같이 Nav.tsx라는 컴포넌트를 제작하여 버튼을 만들고 진행하도록 하겠다.

Header 레이아웃 제작

현재 디자인을 보면 전체적으로 넓이가 제한되어 있는 모습을 볼 수 있는데 이를 위해 우선 전체를 감싸는 컴포넌트를 하나 만들어 width 값을 제한해주도록 하겠다. 필자는 page.tsx에 min-width값을 1200으로 제한해주었다.

앞으로 모든 컴포넌트는 이곳에 import 하여 width값을 모두 1200으로 통일하여 사용 할 수 있을 것이다.

헤더의 레이아웃에 가장 윗 부분을 제작해보자 아래의 이미지를 보면 로고와 로그인 버튼, 텍스트가 보인다 flex를 이용하여 저것과 똑같이 배치해보도록 하자

결과화면

벌써 그럴싸게 레이아웃이 완성 되었다. 그럼 여기서 margin 값과 스타일을 좀더 입혀보도록 하자

스타일을 추가하여 이와같은 결과가 나와다. 아직은 좀 부족해보이기느 하지만 html과 css를 그렇게 깊게 다루지는 않을 예정이라 너무 똑같이 구현하기 위해 시간을 많이 투자 하지는 않고 이렇게 하고 헤더 부분을 끝내도록 하겠다.
